# Student OS - Coding Standards

## General Rules

* Write clean and readable code.
* Use meaningful variable and function names.
* Keep code modular and reusable.
* Add comments where necessary.

## Python/Django Standards

* Follow PEP 8 guidelines.
* Use snake_case for variables and functions.
* Use PascalCase for model and class names.
* Keep models, views, and URLs in separate files.
* Write reusable APIs.

## React Standards

* Use functional components.
* Use PascalCase for component names.
* Keep components reusable.
* Store components inside the src/components folder.

## Tailwind CSS Standards

* Use utility classes for styling.
* Keep class names organized and readable.
* Reuse styles through shared components.

## Git Standards

* Commit frequently.
* Write meaningful commit messages.
* Push code to GitHub regularly.
* Never commit secrets, passwords, or API keys.

## Folder Structure

* accounts → Authentication and user management
* academics → Subjects and class sessions
* notifications → Notification management
* api_app → API endpoints
* frontend → React and Tailwind frontend
